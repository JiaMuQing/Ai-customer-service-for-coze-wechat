# AI 客服管理系统 — 项目计划文档（当前）

## 一、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端 | Node.js + Nest.js | 调用扣子 API、网页聊天、会话落库 |
| 前端 | Vue 3 + Vite + Pinia + Element Plus | 独立访客聊天页、后台登录与会话查询 |
| 数据库 | MySQL 8 | 会话消息等 |
| AI 能力 | 扣子（Coze）API | 知识库在扣子侧；Bot 对话，`conversation_id` 区分多轮 |

- **后台登录**：固定用户名/密码（.env），仅用于 `/admin` 会话查询；**访客聊天页不登录**。
- **企业微信**：当前版本**不接入**，仅网页聊天。

```
┌─────────────────────────────────────────────────────────────────────────┐

```

## 三、数据设计要点

- **conversation_message**：`wecomChatId` 固定 `web`（渠道），`wecomUserId` 为前端生成的 **UUID 访客 ID**（请求头 `X-Web-Visitor-Id`）；`cozeConversationId` 用于扣子多轮。
- **知识库**：在扣子侧维护；本系统不单独做知识库 CRUD 后台（可按需后续加展示）。

---

## 四、核心流程（网页聊天）

1. 用户打开 **`/chat`**（独立界面，与后台无关），前端在 `localStorage` 持久化访客 UUID，请求带 `X-Web-Visitor-Id`。
2. 前端 `POST /chat/message`（**无 JWT**）发送 `content`。
3. 后端按访客 ID 读取最近消息、调用扣子 `/v3/chat`，写回用户与助手消息。
4. `GET /chat/history` 拉取当前访客历史气泡。
5. 运营在 **`/login` → `/admin/session`** 登录后按渠道/访客查库（与访客页分离）。
